# TruyenForge - Hướng Dẫn Xây Dựng Database (Dễ Dùng, Có Code Mẫu)

Tài liệu này viết theo đúng stack hiện tại của dự án: **Vite + Supabase**.
Mục tiêu là để bạn có thể tự dựng hệ thống DB từ đầu, cấu hình local/production, test nhanh, và tránh mất dữ liệu.

---

## 1. Kiến trúc dữ liệu hiện tại (đang dùng trong app)

App đang lưu theo kiểu:
- `user_workspaces`: mỗi user có 1 bản ghi JSON lớn chứa toàn bộ workspace (truyện, nhân vật, prompt, setting...).
- `qa_reports`: lưu lịch sử báo lỗi/chất lượng theo user.

Ưu điểm của kiểu này:
- Dễ triển khai nhanh.
- Đổi cấu trúc dữ liệu trong app linh hoạt hơn.

Nhược điểm:
- Khó query sâu từng chương/từng truyện bằng SQL.

Nếu sau này cần analytics mạnh, bạn có thể tách sang schema chuẩn hóa (normalized) ở bản sau.

## 3. SQL full (copy chạy trực tiếp)

Bạn có thể chạy file hiện có: [`supabase/schema.sql`](/C:/Users/phand/Downloads/ai/truyentudoapp/supabase/schema.sql)

Hoặc copy nguyên khối dưới đây vào SQL Editor:

```sql
create table if not exists public.user_workspaces (
  user_id text primary key,
  payload jsonb not null default '{}'::jsonb,
  updated_at timestamptz not null default timezone('utc', now())
);

alter table public.user_workspaces enable row level security;

create policy "user_workspaces_select_own"
on public.user_workspaces
for select
to authenticated
using (auth.uid()::text = user_id);

create policy "user_workspaces_insert_own"
on public.user_workspaces
for insert
to authenticated
with check (auth.uid()::text = user_id);

create policy "user_workspaces_update_own"
on public.user_workspaces
for update
to authenticated
using (auth.uid()::text = user_id)
with check (auth.uid()::text = user_id);

create table if not exists public.qa_reports (
  id bigint generated by default as identity primary key,
  author_id text not null,
  text_preview text not null default '',
  issue_count integer not null default 0,
  payload jsonb not null default '{}'::jsonb,
  created_at timestamptz not null default timezone('utc', now())
);

create index if not exists qa_reports_author_id_created_at_idx
  on public.qa_reports (author_id, created_at desc);

alter table public.qa_reports enable row level security;

create policy "qa_reports_select_own"
on public.qa_reports
for select
to authenticated
using (auth.uid()::text = author_id);

create policy "qa_reports_insert_own"
on public.qa_reports
for insert
to authenticated
with check (auth.uid()::text = author_id);
```

Tham chiếu file mẫu: [`.env.example`](/C:/Users/phand/Downloads/ai/truyentudoapp/.env.example)

## 5. Code đang dùng để kết nối DB

### 5.1 Supabase client

File: [`src/supabaseClient.ts`](/C:/Users/phand/Downloads/ai/truyentudoapp/src/supabaseClient.ts)

Điểm chính:
- Tạo client từ `VITE_SUPABASE_URL` + `VITE_SUPABASE_ANON_KEY`.
- Nếu thiếu biến thì `supabase = null` để app báo cấu hình thiếu.

### 5.2 Đọc/ghi workspace

File: [`src/supabaseWorkspace.ts`](/C:/Users/phand/Downloads/ai/truyentudoapp/src/supabaseWorkspace.ts)

Các hàm quan trọng:
- `loadServerWorkspace(userId)`
- `saveServerWorkspace(userId, payload)`
- `saveQaReport(userId, report)`

Mẫu dùng:

```ts
import { loadServerWorkspace, saveServerWorkspace } from './supabaseWorkspace';

const data = await loadServerWorkspace<MyWorkspace>(userId);

await saveServerWorkspace(userId, {
  ...workspace,
  updatedAt: new Date().toISOString(),

```

## 7. Quy tắc chống mất dữ liệu (khuyên dùng)

- Không phụ thuộc mỗi autosync nền.
- Mỗi thao tác quan trọng nên có snapshot cục bộ.
- Định kỳ đẩy backup JSON lên Drive.
- Trước khi restore, luôn tạo 1 snapshot an toàn hiện tại.

Luồng an toàn:
1. Local snapshot.
2. Save lên Supabase.
3. Push bản backup lên Drive.

## 8. Lỗi thường gặp và cách xử lý nhanh

### Lỗi: `Supabase chưa được cấu hình đầy đủ`
- Thiếu `VITE_SUPABASE_URL` hoặc `VITE_SUPABASE_ANON_KEY`.
- Kiểm tra local `.env` và biến môi trường trên Vercel.

### Lỗi: `new row violates row-level security policy`
- `user_id` hoặc `author_id` không khớp `auth.uid()`.
- Đăng nhập lại, kiểm tra policy và payload gửi lên.

### Lỗi: `relation "user_workspaces" does not exist`
- Chưa chạy `schema.sql` trên project Supabase đang dùng.
- Hoặc sai tên table trong biến `VITE_SUPABASE_WORKSPACES_TABLE`.

### Lỗi: Vercel có biến env nhưng app vẫn báo thiếu
- Bạn chưa redeploy sau khi thêm env.
- Hoặc thêm biến ở nhầm Environment scope.

## 9. Nâng cấp lên schema chuẩn hóa (khi nào cần)

Chỉ nên tách bảng `stories`, `chapters`, `characters`, `translation_memory` khi:
- Cần filter/sort/query dữ liệu lớn bằng SQL.

- Cần search full-text trực tiếp trên DB.

Khi đó bạn giữ `user_workspaces` làm bản tương thích tạm, rồi migrate dần theo từng module.
